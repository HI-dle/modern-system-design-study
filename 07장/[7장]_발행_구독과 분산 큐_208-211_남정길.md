## 7.4 카프카 스트림


카프카 스트림은 아파치 카프카 위에서 데이터를 처리하려고 설계된 도구이다.

### 카프카 스트림의 특징 

- 카프카와 완전한 통합

    카프카 스트림은 카프카와 완전히 통합되어 카프카 토픽에서 데이터를 자연스럽게 읽고 쓰는것이 가능하고 이 과정에서 카프카의 파티션 및 복제 기능을 활용해 장애 허용성과 확정성을 높일 수 있다.
- 고수준 DSL 제공

    Kafka Streams는 스트림 처리를 쉽게 개발할 수 있도록 고수준 DSL(Domain Specific Language)을 제공한다.

    대표적으로 다음과 같은 연산을 지원한다.

    `map`,`filter`,`join`,`aggregate`

    이를 통해 복잡한 데이터 처리 로직도 선언형 방식으로 간단하게 표현할 수 있다.

- 저수준 Processor API 제공

    Processor API는 데이터 처리 구조를 세밀하게 제어해야 할 때 사용한다.
    이를 통해 개발자는 처리 흐름 직접 설계,커스텀 로직 구현,노드 단위 처리 구성등을 수행할 수 있다.

- 상태 저장(Stateful) 처리 지원

    즉 이전 데이터를 저장해 두었다가 이후 처리 과정에서 활용할 수 있다.

    예를 들어 사용자별 누적 주문 금액,최근 10분 평균값,세션 기반 처리등과 같은 기능을 구현할 수 있다.

    이를 위해 Kafka Streams는 상태 저장소(State Store)를 제공하며, 애플리케이션 내부에서 상태를 저장하고 조회할 수 있도록 지원한다.

- 정확히 한번(Exactly-Once) 처리 지원

    이는 실패하더라도 데이터를 중복 없이 정확히 한 번만 처리 하도록 보장하는 기능이다.

    카프카의 트랜잭션 기능을 활용하여 데이터 중복 처리나 손실을 방지하고 데이터 일관성을 유지할 수 있다.

- 확장성과 장애 허용성

    Kafka Streams 애플리케이션은 여러 인스턴스를 추가하여 수평 확장이 가능하다.

    카프카가 알아서 파티션을 나누어 인스턴스간 작업을 분산 처리하기 때문에 서버를 추가하면 처리량을 자연스럽게 늘릴 수 있다.

    또한 일부 노드에 장애가 발생하더라도 장애 대응(failover) 메커니즘을 통해 다른 인스턴스가 작업을 이어받아 처리할 수 있어 안정적으로 동작한다.


### 스트림 처리 토폴로지(Topology)

Kafka Streams에서는 스트림 처리 애플리케이션을 토폴로지(Topology) 형태로 구성한다.

토폴로지는 스트림 프로세서(Node)와 스트림(Edge)로 구성된 그래프이다


**Source** : 입력 토픽(카프카에서 데이터 읽기)<br>
**Processor** :	프로세서(데이터 가공 및 처리)<br>
**Sink** :	출력 토픽(처리 결과 저장)<br>

Source → Processor → Sink 이 흐름으로 입력 데이터를 처리하여 출력 스트림을 생성한다.

즉 데이터를 입력받아 처리한 뒤 결과를 저장하는 전체 흐름을 토폴로지라고 한다.

#### 카프카 스트림을 활용한 사례

카프카 스트림은 실시간 데이터 처리에 적합하며, 다양한 시스템에서 스트림 기반 데이터 처리를 구현할 때 활용할 수 있다.

- 실시간 데이터 분석

    스트리밍 데이터를 실시간으로 분석하여 중요한 정보를 추출하거나 이상 징후를 감지할 수 있다.
    즉 데이터가 저장된 이후 분석하는 것이 아니라, 데이터가 들어오는 즉시 처리하여 빠르게 대응할 수 있다는 장점이 있다.
    * 사용자 행동 분석
    * 실시간 로그 분석

- 이벤트 기반 아키텍처

    카프카 스트림은 실시간 이벤트를 처리하고 반응하는 시스템을 구축하는 데 활용할 수 있다.
    각 서비스는 필요한 이벤트만 구독하여 처리할 수 있기 때문에 서비스 간 결합도를 낮추고 유연한 구조를 만들 수 있다.
    * 마이크로서비스 간 통신
    * 서비스 간 비동기 이벤트 처리
    * 이벤트 기반 데이터 흐름 구성

- 데이터 보강 및 변환

    스트리밍 데이터에 외부 데이터를 추가하여 정보를 보강하거나 후속 시스템에서 활용할 수 있도록 필요한 형태로 변환할 수 있다.
    이를 통해 단순 데이터 전달이 아닌 가공된 데이터를 실시간으로 제공할 수 있다.
    * 사용자 정보 조인
    * 데이터 포맷 변환

- 부정행위 감지

    실시간으로 거래나 사용자 행동을 분석하여 비정상적인 패턴이나 부정행위를 탐지할 수 있다.
    실시간 스트림 처리 기반이기 때문에 이상 상황 발생 시 즉시 대응할 수 있다는 장점이 있다.
    * 비정상 결제 탐지
    * 로그인 이상 패턴 감지
    * 봇 활동 탐지

- 스트리밍 ETL

    카프카 스트림은 스트리밍 데이터를 실시간으로 추출(Extract),변환(Transform),적재(Load)하는 ETL 처리에도 활용할 수 있다. 이를 통해 지속적으로 데이터를 통합하고 처리할 수 있으며, 배치 처리보다 빠르게 데이터를 활용할 수 있다.



#### 카프카 스트림의 장점

카프카 스트림을 활용하면 효과적이고 유연한 스트림 처리 애플리케이션을 구축할 수 있다.
특히 카프카 기반 아키텍처와 자연스럽게 통합되기 때문에 실시간 데이터 처리에 적합한 도구로 평가받고 있다.


> **아마존 키네시스** <br>
>카프카는 강력한 스트림 처리 플랫폼이지만, 분산 시스템 운영에는 복잡한 관리와 운영 작업이 필요하다.
>
>이러한 흐름 속에서 아마존 키네시스는 클라우드 기반 완전 관리형 서비스로 등장하였다.
>
>키네시스는 스트리밍 데이터 처리 과정에서 발생하는 다양한 운영 부담을 줄여주며, 클라우드 환경에서 손쉽게 스트림 처리 시스템을 구축할 수 있도록 지원한다.<br>
>또한 카프카와 비교했을 때의 차이점과 서로 보완 가능한 부분들을 이해하는 것이 중요하며
>이를 통해 현대 데이터 중심 애플리케이션 환경에서 키네시스가 어떤 역할을 수행하는지 살펴볼 수있다.


### 7.5 키네시스

키네시스(Kinesis)는 AWS에서 제공하는 클라우드 기반 스트리밍 데이터 처리 서비스이다.

대규모 데이터 스트림을 실시간으로 처리할 수 있도록 설계되었으며, 실시간 분석, 로그 처리, 이벤트 처리, 모니터링 등에 활용된다.

키네시스는 여러 구성 요소를 통해 스트리밍 데이터를 수집·저장·분석한다.

- **키네시스 데이터 스트림(Kinesis Data Streams)**

    키네시스 데이터 스트림은 실시간으로 대규모 데이터를 수집하고 처리하는 핵심 서비스이다.

    로그 데이터나 실시간 이벤트 데이터를 지속적으로 수집할 수 있으며, 여러 소스의 데이터를 동시에 처리할 수 있어 즉각적인 데이터 분석과 실시간 처리가 필요한 환경에 적합하다.

- **키네시스 데이터 파이어호스(Kinesis Data Firehose)**

    키네시스 데이터 파이어호스는 키네시스 데이터 스트림과 연계해 스트리밍 데이터를 AWS 서비스 및 외부 저장소로 자동 적재하는 서비스이다.

    필요에 따라 데이터를 변환하여 저장할 수 있으며, Amazon S3, Amazon Redshift, Amazon ElasticSearch, Splunk과 같은 저장소와 연동 가능하다.

    이를 통해 데이터 저장 및 분석 환경을 쉽게 구축할 수 있다.

- **키네시스 데이터 애널리틱스(Kinesis Data Analytics)**

    키네시스 데이터 애널리틱스는 SQL 기반으로 스트리밍 데이터를 실시간 분석할 수 있는 서비스이다.

    표준 SQL 쿼리를 사용하여 스트림 데이터를 직접 처리하며 특히 키네시스 데이터 스트림과 함께 사용시 데이터를 배치 처리하거나 별도로 내보내지 않고 즉시 분석할 수 있어 실시간 집계나 패턴 분석 등에 활용된다.

#### 키네시스의 특징

키네시스는 데이터의 수집, 저장, 처리, 분석 과정을 모두 지원하는 실시간 스트리밍 플랫폼이다.

특히 AWS 환경과 자연스럽게 통합되기 때문에 클라우드 기반 스트림 처리 시스템 구축에 적합하다.

### 7.6 요약

이 장에서는 분산 시스템의 핵심 개념인 발행/구독 시스템과 분산 큐 시스템을 학습하였다.

분산 큐를 통해 확장성, 안정성, 메시지 무결성을 보장하는 방법과, 발행/구독 시스템이 서비스 간 결합도를 낮추고 유연성을 높이는 역할을 이해하였다.

또한 마이크로서비스와의 연계 방식, 그리고 Apache의 Apache Kafka와 Amazon의 Amazon Kinesis의 핵심 개념과 동작 원리도 살펴보았다.

이를 통해 효율적이고 확장 가능한 분산 시스템 설계의 기본 개념을 익혔으며, 다음 장에서는 API 설계, 보안, 메트릭 등 분산 시스템의 핵심 요소를 다룬다.
