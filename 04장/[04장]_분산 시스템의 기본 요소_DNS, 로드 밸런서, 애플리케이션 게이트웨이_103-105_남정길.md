
**권한 있는 네임 서버란?**

영어 원문 'Authoritative'를 직역한 것

해당 도메인에 대한 **최종 결정권을 가진 서버**를 의미한다.<br>
쉽게 말해 해당 도메인의 IP 주소와 같은 정보를 공식적으로 제공할 수 있는 서버

자연스럽게 표현하려면 '공인 서버'나 '공식 네임 서버'처럼 바꿀 수도 있지만,
기술 문서에서는 일반적으로 '권한 있는 네임 서버'라는 번역 표현을 많이 사용

권한 있는 네임 서버는 최종적으로 도메인 이름을 IP 주소로 변환하는 역할을 하는 DNS 서버로, 가장 정확한 도메인 정
보를 가지고 있는 서버라고 이해하면 된다.


**리졸버가 무엇인가요?**

'리졸버(resolver)'는 DNS 시스템에서 사용자가 입력한 도메인 이름을 IP 주소로 변환하는 역할을 하는 요소

쉽게 말해 리졸버는 도메인 이름을 입력하면 이 이름에 해당하는 IP 주소를 찾아 주는 중개자 역할을 한다.<br>
보통 사용자의 인터넷 서비스 제공업체(ISP)나 회사 네트워크의 로컬 DNS 서버가 리졸버 역할을 수행하여 빠르고 효율적
인 인터넷 연결을 지원한다.

### 4.1.1 DNS 쿼리 

DNS 쿼리는 기본적으로 `반복적 쿼리`와 `재귀적 쿼리` 두 가지 방식을 사용하며 이 두 방식은 컴퓨
터가 www.google.com 같은 웹 사이트 이름에 대응하는 IP 주소를 찾는 방법을 뜻한다.

**반복적 쿼리**
반복적 쿼리에서는 컴퓨터가 로컬 DNS 리졸버로 모든 과정을 직접 처리한다. DNS 체계의 최상위에 있
는 루트 서버에서 시작하여 최상위 도메인 서버(.com, .org 등)를 거쳐 최종적으로 사용자가 방문하려는
웹 사이트의 권한 있는 네임 서버에 직접 쿼리를 보내 IP 주소를 찾는다.

어떤 웹 사이트로 접속하면 컴퓨터는 루트 서버에서 시작해서 최상위 도메인 서버, 웹 사이트의
권한 있는 네임 서버와 순차적으로 직접 통신한다.

이 단계를 하나씩 거치며 원하는 IP 주소를 찾는다.

**DNS 재귀적 쿼리**

컴퓨터가 로컬 DNS 리졸버에 원하는 웹 사이트의 IP 주소를 요청하면 로컬 리졸버가 이후 모든 과정을 처리하고
루트 서버에서 시작해서 최상위 도메인 서버, 웹 사이트의 권한 있는 네임 서버까지 순차적으로 쿼리를 보내며 IP 주소를 찾는다
즉, 사용자는 로컬 리졸버에 IP 주소를 요청하고, 이후 리졸버가 필요한 IP 주소를 사용자에게 전달한다.


**반복적 쿼리와 재귀적 쿼리의 차이**

반복적 쿼리는 **사용자(컴퓨터)가 여러 곳에 직접 물어보며 답을 찾아가는 방식**<br>
예를 들어 한지훈이 길을 찾는데 A한테 물어보고 모른다하면 B한테 가서 묻는식<br>
사용자가 직접 각 단계 마다 다른 서버에 요청을 보내서 답을 찾는 구조 

재귀적 쿼리는 **사용자가 한 번만 요청을 보내면 그 뒤는 리졸버가 알아서 모든 과정을 처리하는 방식**
마치 도서관에 가서 “이 책을 찾고 싶어요."라고 말하면 도서관 사서가 직접 책을 찾아서 가져다주는 것과 비슷

일반적으로 우리가 쓰는 인터넷에서는 이 재귀적 쿼리 방식을 사용한다
