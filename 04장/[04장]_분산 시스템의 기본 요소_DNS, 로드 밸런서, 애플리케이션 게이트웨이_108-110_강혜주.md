## DNS 캐싱

웹 사이트 이름을 입력하면 거의 즉시 해당 웹 페이지가 나타난다. 이렇게 빠른 인터페이스를 제공할 수 있는 것은 바로 DNS 캐싱 덕분이다.

### 개념

캐싱이란 자주 접근하는 리소스 레코드를 임시로 저장하는 것이다. 리소스 레코드는 DNS 데이터베이스에서 이름과 값을 연결하는 데이터 단위이다.

### 장점

- DNS 계층 구조로 쿼리하지 않고 로컬에서 답을 제공함으로써 사용자 응답 시간을 단축한다.
- 불필요한 쿼리를 줄여 네트워크 트래픽을 감소시킨다.


여러 단계에서 캐싱을 적용하면 전체 인터넷의 쿼리를 처리하는 DNS 인프라 부하를 크게 줄일 수 있다.

### 사용위치

- 사용자의 웹 브라우저
- 운영 체제
- 사용자 네트워크 내 로컬 네임 서버
- ISP(인터넷 서비스 제공업체. 보통 통신사)의 리졸버

따라서 DNS 리소스 레코드를 여러 단계에서 캐싱하면 로컬에서 바로 답을 제공할 수 있어 사용자에게 더 빠른 응답을 줄 수 있고, DNS 시스템의 트래픽도 줄어든다.
특히 여러 단계(브라우저, 운영 체제, 로컬 네임 서버, ISP 리졸버)에서 캐싱을 하면 이 효과는 더욱 커진다.

> 하지만 DNS 값을 변경해도 사용자마다 캐시 상태가 달라서 즉시 반영처럼 보이지 않을 수 있다.

## 4.2 DNS의 확장성, 신뢰성, 일관성

대규모 시스템(특히 DNS)처럼 방대한 시스템을 논할 때 자주 언급되는 세 가지 핵심 개념이 있다. 확장성, 신뢰성, 일관성이다. 이 세 가지는 견고한 시스템을 지탱하는 요소이다.

---

## 4.2.1 확장성

DNS는 루트 서버, 최상위 도메인 서버, 권한 있는 네임 서버로 구성된 계층적 구조로 높은 확장성을 지닌다.
- 전 세계에 분산된 루트 서버 인스턴스 약 1000개가 요청의 시작점인 사용자 쿼리를 처리하고 이를 최상위 도메인 서버로 연결한다.
- 이후 최상위 도메인 서버는 도메인 소유 조직(예: google.com에서는 구글)이 관리하는 권한 있는 네임 서버로 연결한다.
- 이런 계층 구조 덕분에 DNS는 단계별로 부하를 분산시켜 전 세계에서 발생하는 쿼리 수십억 건을 효율적으로 처리할 수 있는 확장성을 갖추게 된다.

DNS의 분산 구조는 시스템 확장성을 높이는 데 큰 역할을 한다. 전 세계 여러 곳에 중복된 DNS 서버가 배치되어 있어 사용자 요청에 빠르게 응답하며 지연 시간을 줄일 수 있다. 한 DNS 서버가 과부하되거나 사용 불가능한 상태가 되더라도 다른 서버가 요청을 대신 처리함으로써 시스템의 안정성과 확장성이 더욱 강화된다.

>계층 구조는 위 단계가 모든 정답을 알고 있다는 의미가 아니다. 역할을 나눠 처리하는 구조이다. “권한 있는 네임 서버”가 해당 도메인의 레코드를 최종적으로 제공하는 주체이다.

---

## 4.2.2 신뢰성

DNS는 다양한 방법으로 신뢰성을 높게 유지한다. 웹 브라우저, 운영 체제, 로컬 네트워크, ISP 리졸버 등 여러 단계에서 캐싱을 사용하므로 **일부 DNS 서버가 일시적으로 응답하지 못하더라도 사용자에게 필요한 정보를 빠르게 전달할 수 있다.**

또 전 세계에 분산된 중복 DNS 서버가 사용자 요청에 신속히 응답하여 시스템 안정성을 높인다.

대부분의 쿼리와 응답은 신뢰성이 낮은 UDP 프로토콜을 사용하지만, DNS는 응답이 없을 때 요청을 재전송하는 방식으로 이를 보완한다. 이 방식 덕분에 DNS는 핵심 기능을 안정적으로 수행할 수 있는 신뢰성을 확보하고 있다.

---

## 4.2.3 일관성

DNS는 강한 일관성을 유지하는 대신 높은 성능을 목표로 설계되었다. 캐싱된 레코드의 유효 기간(TTL) 설정과 점진적 업데이트를 반영하여 시간이 지나면 모든 서버가 일관된 데이터를 가지는 최종 일관성을 적용했다. DNS 시스템 내에서 업데이트가 모든 서버에 반영되는 데는 몇 초에서 며칠이 걸릴 수 있다.

그렇지만 DNS에서는 여전히 성능이 최우선이다. 일관성을 보장하는 방법은 여럿이지만, 성능을 크게 저하시시키지 않는 수준에서만 일관성을 유지하도록 설계되어 있다.
