  - 주요 서비스의 세부 설계에 집중해서 확장성, 신뢰성, 성능을 확보 방법을 알아본다.

# 11.1 기능적 요구 사항

## X가 갖추어야 할 핵심 기능 종합 정리
- 사용자 등록 및 인증
  - 사용자 이름, 이메일, 비밀번호 등 필수 정보를 입력하여 `계정 생성`
  - 사용자가 로그인할 때 입력하는 자격 증명은 안전하게 저장되고, 정확한 `인증` 절차 필요
  - 사용자 `세션 관리`로 서비스에 원활한 접근 가능
- 트윗 생성
  - 최대 글자 수(예 280자)로 정해진 `짧은 메시지` 작성
  - 트윗에는 텍스트, 해시태그, 다른 사용자를 언급하는 멘션이나 이미지, 동영상 같은 `미디어 첨부`
  - 트윗당 입력 가능한 `글자 수 제한` 준수
  - 트윗 `빠른 저장 및 조회`
- 팔로우/언팔로우
  - 다른 사용자를 `팔로우`해서 해당 사용자의 트윗을 보는 타임라인 기능
  - 사용자 `관계를 나타내는 팔로우 관리`할 수 있어야 합니다.
  - 언제든지 다른 사용자를 `언팔로우`해서 해당 사용자의 트윗이 자신의 타임라인에 반영.
- 타임라인
  - 타임라인은 사용자가 `팔로우하는 계정의 트윗을 시간 순대로 조회`
  - 사용자의 `트윗 생성 시간과 선호도를 반영`하여 각 사용자에게 맞는 내용을 효율적으로 생성해서 타임라인에 표시
  - `실시간 업데이트를 지원`하여 사용자가 새로운 트윗을 바로 확인
- 검색
  - `키워드, 해시태그, 사용자 이름을 기반` 트윗 또는 다른 사용자 검색
  - 검색 기능은 `연관성, 인기, 최신성을 고려`하여 가장 적절하고 `정확한` 결과
  - 트윗과 사용자 데이터를 효율적으로 `인덱싱`하고 `저장`하여 빠르고 확장 가능한 검색이 가능
- 리트윗과 좋아요 기능
  - 다른 사용자의 트윗을 자신의 팔로워에게 `공유`할 수 있는 리트윗 기능
  - 리트윗은 원본 트윗의 `메타데이터`와 출처 정보 유지 및 효율적 처리
  - 트윗에 `공감`하거나 `좋아요` 기능
- 다이렉트 메시지
  - 사용자끼리 서로 개인적으로 `소통`할 수 있도록 다이렉트 메시지 기능
  - 공개 트윗 피드와는 별도
  - 다이렉트 메시지 내에 포함된 `사생활을 보호`할 수 있는 `적절한 접근 제어와 암호화 필요`
  
이런 요구 사항을 충족하면 마이크로블로깅을 하고, 다른 사람과 소통하며, 실시간으로 원하는 콘텐츠를 즐길 수 있다.

다음으로는 서비스가 확장성, 안정성 및 성능을 유지하면서도 앞서 언급한 기능적 요구 사항을 실현할 수 있도록 하는 비기능적 요구 사항을 알아본다.
