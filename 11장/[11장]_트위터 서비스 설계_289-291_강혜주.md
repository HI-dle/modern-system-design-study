# 11.5.1 마이크로서비스 아키텍처

마이크로서비스 아키텍처를 적용하면 각 요소가 더 모듈화되고, 확장성과 유지 보수성도 크게 향상된다.

각 마이크로서비스는 특정 기능이나 도메인을 담당하며, X에서 핵심 역할을 하는 주요 마이크로서비스는 다음과 같다.

![그림_11_2.png](image/%EA%B7%B8%EB%A6%BC_11_2.png)

## 트윗 서비스

- 트윗 생성, 조회, 삭제를 처리한다.
- 트윗 데이터를 데이터베이스에 저장하고, 미디어 파일은 객체 저장소에 보관한다.
- 새로운 트윗을 메시지 큐에 전달하여 다른 서비스가 처리할 수 있도록 한다.

## 사용자 서비스

- 사용자 등록, 인증, 프로필 정보를 관리한다.
- 사용자 데이터를 데이터베이스에 저장한다.
- 팔로우와 언팔로우 기능을 처리하며, 팔로워-팔로잉 관계를 유지한다.

## 타임라인 서비스

- 사용자가 팔로우한 계정의 트윗을 모아 타임라인을 생성하고 제공한다.
- 메시지 큐에서 새로운 트윗을 받아 사용자 타임라인을 최신 상태로 업데이트한다.
- 타임라인 데이터를 캐시에 저장하여 빠르게 조회할 수 있도록 처리한다.

## 검색 서비스

- 키워드, 해시태그, 기타 조건을 기준으로 트윗과 사용자를 검색할 수 있도록 한다.
- 트윗과 사용자 데이터를 효율적으로 검색할 수 있도록 인덱싱한다.
- 검색 결과를 연관성이나 인기도에 따라 순위별로 정렬하여 제공한다.

# 공통 구성 요소

서비스 성능을 높이고 확장성과 안정성을 확보하려면 여러 마이크로서비스에서 재사용할 수 있는 공통 소프트웨어 모듈과 기능이 필요하다.

여기에는 캐싱, 데이터 모델 설계, 메시지 큐 활용, 데이터 동기화 방식, 모니터링 및 로깅, 보안과 프라이버시 같은 요소들이 포함된다.

## 캐싱

- Redis 같은 분산 캐시 레이어를 사용하면 백엔드 서비스의 부하를 줄이고 성능을 끌어올릴 수 있다.
- 캐싱 레이어에는 사용자 프로필, 인기 트윗, 타임라인 데이터처럼 자주 조회하는 데이터를 저장한다.
- 이를 통해 빠르게 찾을 수 있으며, 데이터베이스 요청 수도 줄일 수 있다.

## 데이터베이스

- X 같은 서비스는 Apache Cassandra나 Amazon DynamoDB 같은 분산 데이터베이스를 사용할 수 있다.
- 트윗, 사용자 정보, 팔로우/팔로워 관계, 좋아요, 리트윗 등 정형 데이터를 저장한다.
- 데이터베이스는 높은 쓰기 처리량과 낮은 지연 시간을 지원해야 한다.
- 데이터를 여러 노드에 분산하고 가용성을 확보하려면 샤딩과 복제 같은 기법을 활용할 수 있다.


## 객체 저장소

- 이미지와 동영상 등 미디어 파일은 Amazon S3 같은 객체 저장소에 보관한다.
- 객체 저장소는 확장성과 안정성이 좋아 대용량 파일 저장에 적합하다.
- 사용자가 데이터를 효율적으로 가져오거나 저장할 수 있도록 도와준다.

## 메시지 큐

- 마이크로서비스 간 비동기 통신을 위해 Kafka 같은 메시지 큐를 사용할 수 있다.
- 사용자가 새로운 트윗을 작성하면 해당 트윗은 메시지 큐에 등록된다.
- 이후 타임라인 서비스와 검색 서비스가 메시지를 받아 필요한 데이터를 업데이트한다.
- 메시지 큐를 사용하면 서비스 간 결합도를 낮출 수 있다.
- 트윗 처리 작업을 확장 가능하게 만들 수 있다.

## 실시간 업데이트

- WebSocket을 사용하면 사용자가 실시간으로 업데이트를 받을 수 있다.
- 클라이언트와 서버를 연결해 새로운 트윗, 알림, 팔로워 추가 같은 이벤트가 발생했을 때 바로 전달할 수 있다.
- 서버가 WebSocket을 통해 관련 클라이언트에 알림을 전달하면 즉각적으로 업데이트가 가능하다.

## 모니터링 및 로깅

- 시스템의 상태와 성능을 관리하려면 체계적인 모니터링과 로깅 메커니즘이 필요하다.
- Prometheus와 Grafana 같은 도구로 사용량, 요청 지연 시간, 오류율, 자원 사용량 등 주요 지표를 수집하고 시각화할 수 있다.
- ELK Stack 같은 중앙 집중형 로깅 시스템을 사용하면 여러 구성 요소의 로그를 통합하고 분석할 수 있다.



## 보안 및 프라이버시

- 시스템 전반에서 보안과 프라이버시를 강화하는 조치를 적용해야 한다.
- 사용자 인증과 권한 관리는 OAuth 같은 안전한 프로토콜을 사용할 수 있다.
- 비밀번호처럼 민감한 데이터는 해시 처리 후 안전하게 저장한다.
- 데이터 전송 중이나 저장 중에는 암호화를 적용해 데이터를 보호해야 한다.
- 서비스 남용을 방지하기 위해 공정한 사용을 보장하고, 요청 속도 제한과 트래픽 제어 메커니즘도 도입해야 한다.



# 정리

지금까지 X 서비스의 주요 구성 요소와 각 요소 간 상호 작용을 전체적으로 정리해 보았다.

X 서비스의 규모에 맞는 요구 사항을 반영하고, 아키텍처가 확장성과 효율성을 갖출 수 있도록 설계했다.
