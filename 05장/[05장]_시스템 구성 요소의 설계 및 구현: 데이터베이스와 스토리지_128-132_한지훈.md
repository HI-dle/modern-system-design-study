## 5.1 데이터베이스

 - 데이터베이스는 방대한 양의 정보를 체계적이고 일관된 방식으로 저장하고 관리하며 활용할 수 있게한다. 데이터는 종류에 따라 구조화, 반구조화된 데이터일 수 있다.
   - 데이터 구성: 데이터베이스는 데이터를 구조적이고 체계적으로 저장하는 방식이다. 데이터는 테이블, 행, 열 형태로 정리되고 정보를 쉽게 분류하고 접근할 수 있다.
   - 데이터 검색: 데이터베이스는 빠르고 효율적으로 데이터 검색이 가능하다. 복잡한 쿼리를 사용해 원하는 데이터를 추출할 수 있으며 인덱싱 메커니즘으로 데이터를 빠르게 조회할 수 있다.
   - 데이터 무결성: 제약조건, 관계, 검증 규칙을 이용헤 데이터 무결성을 유지한다. 데이터의 신뢰성, 정확성, 일관성이 보장된다.
   - 데이터 보안: 사용자 인증, 권한 부여, 암호화 등 보안 기능을 통해 무단으로 데이터 유출을 막아준다.
   - 데이터 일관성: 트랜잭션 관리와 ACID(원자성, 일관성, 독립성, 지속성) 특성을 이용해 여러 사용자 동시 접근에 대해 일관성을 유지한다.
   - 확장성: 대규모 데이터를 처리하고 데이터 수요가 증가함에 따라 확장할 수 있다. 단일 서버에 리소스를 추가하는 수직 확장과 여러 서버나 노드를 추가하는 수평 확장 모두 가능하다.
   - 다중화, 백업 및 복구: 데이터의 가용성과 신뢰성을 보장하려고 다중화와 백업 매커니즘을 갖추고 있다. 복제와 자동 백업 같은 기능이 포함되며 데이터 손실이나 시스템 장애가 발생할 때 데이터를 백업하고 복구할 수 있는 기능도 포함된다.
   - 복잡한 쿼리: 대규모 데이터셋을 다룰 때 복잡한 쿼리와 통게 작업을 실행할 수 있어 정보 추출과 분석이 가능하다.
   - 데이터 관계: 관계형 데이터베이스에서는 데이터 간 관게가 정의되어 있므ㅡ로 고객 주문, 제품, 재고처럼 서로 연관된 데이터를 효율적으로 관리할 수 있다.
   - 데이터 이력: 데이터 변화를 기록해 과거 데이터를 조회할 수 있다. 감시나 법적 요구 사항을 충족하는 데 도움이 된다.
   - 데이터 분석: 데이터 분석 및 보고서 작성 도구를 지원한다. 데이터 기반 의사 결정을 내리고 운영 상태와 성과 현황을 파악할 수 있다.
   - 데이터 공유: 여러 사용자와 애플리케이션이 동시에 데이터에 접근하고 공유할 수 있도록 지원해 협업과 실시간 작업 환경에서 필수 요소가 된다.
- 다만 하나의 데이터베이스 유형이 모든 시스템과 애플리케이션의 요구 사항을 모두 충족할 수 없기에 각 목적에 맞는 다양한 데이터 베이스를 사용해야한다.

### 5.1.1 데이터베이스 유형
 - 데이터베이스는 관계형과 NoSQL 데이터베이스로 분류할 수 있다. 선택은 애플리케이션이나 시스템의 특정 요구사항에 따라 달라지고 구조, 확장성, 성능 등 요소가 중요 기준이 된다.

### 5.1.2 관게형 데이터베이스

 - 관계형 데이터베이스는 데이터를 행과 열로 구성해 표 형식으로 관리하는 유형이다. 데이터 요소 간 관게를 정의해 효율적으로 데이터 저장과 검색을 한다.
   - 테이블: 데이터를 테이블 형태로 표현한다. 각 테이블은 고객, 제품, 주문 등 개체나 개념을 나타내고 행과 열로 나눈다.
   - 행: 레코드, 튜플이라 하며 고유한 데이터 항목을 나타낸다. 고객 테이블에선 각 행이 개별 고객을 나타낸다.
   - 열: 속성 또는 필드라 하고 테이블에 저장할 수 있는 데이터 유형을 의미한다. 각 열은 이름과 데이터 타입을 가진다.
   - 키: 관계형 데이터베이스는 키를 통해 테이블간 관계를 설정한다. 기본키는 테이블의 각 행을 고유하게 구별하고 외래 키는 한 테이블에서 다른 테이블의 기본키를 참조해 두 테이블간 관계를 형성한다.
   - 정규화: 데이터 중복을 제거하고 데이터 무결성을 높인다. 이 과정에서 테이블을 나누고 서로 연결해 중복을 줄이고 데이터 일관성을 유지한다.
   - SQL: DB와 상호작용하는 언어로 생성, 조회, 수정, 삭제 등을 할 수 있고 테이블 구조 정의 및 테이블간 관계를 설정하는 표준 방식이다.
   - ACID: 관계형 데이터베이스는 데이터의 원자성, 일관성, 독립성, 지속성을 보장하는 특성을 갖추고 있다.
   - 트랜잭션: 여러 SQL 작업을 하나의 단위로 묶어 처리하는 기능으로 롤백과 커밋을 통해 동작해 데이터 일관성을 유지한다.
 - 대표적인 RDBMS
   - MySQL
   - PostgreSQL
   - Oracle Database
   - Microsoft SQL Server
   - SQLite
   - IBM Db2
 - RDB는 금융, 재고 관리, 고객 관계 관리 등 데이터 일관성, 구조, 신뢰성이 중요한 분야에 주로 사용된다. 

### 5.1.3 비관계형 데이터베이스
- 비관계형 데이터베이스, 흔히 NoSQL 데이터베이스라고 하는 이 시스템은 일반적인 관계형 데이터베이스 모델에서 벗어난 데이터베이스 관리 시스템이다. 테이블, 행, 열을 사용하는 관계형 데이터베이스와 달리 비관계형 데이터베이스는 데이터 저장과 검색에서 더 유연한 방식을 사용하며, 비정형 또는 반정형 데이터를 처리하도록 설계되어 다양한 애플리케이션과 용도에 적합하다.
 - 비관계형 데이터베이스의 특징
   - 키-값 저장소: 고유한 키와 데이터 값을 연관 짓는 방식으로, 단순한 데이터 검색에 매우 효율적이지만 복잡한 쿼리에는 적합하지 않다. 대표적인 예로는 레디스, 아마존 DynamoDB, Riak 등이 있다.
   - 문서 지향 데이터베이스: JSON이나 XML 같은 형식의 문서로 데이터를 저장하며, 주로 웹 애플리케이션에서 많이 사용한다. 대표적인 예로 MongoDB, CouchDB, RavenDB가 있다.
   - 컬럼 패밀리 데이터베이스: 관련된 데이터를 컬럼 패밀리라는 그룹으로 구성한다. 수평 확장에 유리한 특성이 있다. 아파치 카산드라, HBase, ScyllaDB 등이 이에 해당한다.
   - 그래프 기반 데이터베이스: 복잡한 관계를 표현하는 데이터를 위해 설계했다. 데이터 간 관계를 나타내고 탐색하는 데 그래프 구조를 사용한다. 대표적인 예로 Neo4j, 아마존 Neptune, OrientDB가 있다.
 - 비관계형 데이터베이스의 주요 장점으로는 대용량 데이터를 처리할 수 있는 능력, 확장성, 변화하는 데이터 구조와 요구 사항에 유연하게 대응할 수 있는 점 등이 있다. 하지만 이런 장점에는 강력한 데이터 일관성에 대한 요구가 낮아지고 더욱 신중한 데이터 모델링과 인덱싱 설계가 필요하다는 단점이 따른다.
