## 5.12 관계형 모델링과 그래프 모델링

- 3 노드의 관계를 데이터로 표현해야 한다고 가정할때 크게 관계형 모델링과 그래프 모델링 중 선택할 수 있다. 관계형 모델링 방식으로 데이터를 저장하는 방법은 다시 2가지로 나눌 수 있다.

 - 관계를 단방향으로 저장
   - 관계(에지)의 시작점과 도착점을 하나의 행으로 저장한다 (a)에서 표현하는 방식으로 모든 관계를 이런 형태로 저장할 수 있다. 특정 사용자가 팔로우 하는 대상(도착점)을 찾으려면 인덱스로 해당 사용자의 시작점을 찾아야 한다.

- 관계를 양방향으로 저장
  - 관계의 출발점과 도착점을 한 번씩 번갈아 가며 저장하고 방향을 나타내는 추가 열을 더 보탠다. 이는 (b)에서 보인다.

### 5.12.1 그래프 모델링

![그림5-20.png](image/%EA%B7%B8%EB%A6%BC5-20.png)

 - 데이터를 그래프 모델링 방식으로 저장하는 경우 데이터를 이중 연결 리스트 그래프와 두 테이블로 표현할 수 있다.
 - 위 그림에서 a는 노드와 그 관계를 이중 연결 리스트로 표현한 것으로 노드가 3개 있으므로 연결 리스트도 3개다.
 - b는 노드가 실제로 저장되는 방식을 표현한 것으로 각 레코드에는 해당 노드의 첫번째 관계 ID가 저장된다. 예시로 노드 1과 2의 첫번째 관계는 모두 A이다.
 - c는 관계가 실제로 저장되는 방식을 의미한다. 각 레코드에는 관계의 출발점과 도착점이 저장된다. 이외에도 출발 노드와 도착 노드 각각에서 이전 관계와 다음 관계 ID도 저장된다.

### 5.12.2 기존 그래프에 새로운 노드 추가


 - 이번엔 노드 4를 그래프에 추가하려 한다고 가정할때 노드 2가 노드 4를 팔로우 하는 관계를 추가하려면 다음 그링과 같이 이중 연결 리스트와 두 테이블을 수정해야 한다.

![그림5-21.png](image/%EA%B7%B8%EB%A6%BC5-21.png)

 - 새로운 노드가 추가되었을때 새 노드 ID는 4고 관계 테이블에는 새로운 관계 D가 추가되었다. 새로운 관계 D는 출발점이 2고 도착점이 4이다. 즉, 노드 2가 노드 4를 팔로우하는 구조이다.
 - 노드 테이블엔 nid =4, first_rid = D가 들어 있는 새로운 행을 추가했다.
 - 관계 테이블에 새로운 관계 D를 추가하면서 각 열에 필요한 값을 입력하고 기존 데이터 수정되었다.


### 노드와 관게가 저장되는 방식
- 노드 저장소
  - 데이터베이스에 있는 모든 노드는 노드 저장소 파일에 저장된다. 이때 각 노드 레코드는 15바이트 크기이다.
  - 레코드 구조는 다음과 같다.
    - 첫번쨰 바이트: 사용 중인지 나타냄
    - 다음 4바이트: 노드ID
    - 그다음 1바이트: 첫번째 관계 ID
    - 그다음 1바이트: 첫번째 속성 ID
    - 다음 5바이트: 레이블 저장소
    - 남은 1바이트: 향후 사용 공간
- 관계 저장소
  - 노드와 달리 관계 정보 레코드는 34바이트로 구성되며 구조는 다음과 같다.
    - 출발 노드 ID: 관계 시작점을 나타냄
    - 도착 노드 ID: 관계 끝점을 나타냄
    - 관계 유형 포인터: 관계의 유형 정보를 가리킴
    - 각 출발 노드와 도착 노드에 대한 이전 및 다음 관계 레코드 포인터: 연결된 관계 레코드 간 탐색을 지원


## 5.13 요약

- 데이터 베이스 본질은 데이터를 체계적으로 관리하고, 필요한 정보를 빠르게 조회하며, 데이터 무결성을 유지하는 역할을 한다.
- 다양한 시스템과 애플리케이션에서 데이터를 효율적으로 처리할 수 있게 하고, 의사 결정을 뒷받침하는 데 활용된다.

### 데이터베이스 유형
- **관계형 데이터베이스**
  - 테이블 형태의 구조와 강력한 데이터 무결성을 기반으로, 구조화된 데이터와 복잡한 데이터를 다루는 애플리케이션에 적합하다
  - 대표 예시: MySQL, PostgreSQL, Oracle.
- **비관계형 데이터베이스**
  - 유연성과 확장성이 강점이다.
  - **키-값 저장소**: 아마존 DynamoDB — 효율적인 데이터 저장 및 조회.
  - **컬럼 패밀리 데이터베이스**: 아파치 카산드라, HBase — 대규모 데이터 처리와 쓰기 작업에 강점이 있다.
  - **그래프 기반 데이터베이스**: Neo4j — 복잡하게 얽힌 데이터 관계 표현 및 탐색에 뛰어나며 소셜 네트워크, 추천 시스템, 지식 그래프 등에서 활용한다.
