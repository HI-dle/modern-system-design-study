# 3장 분산 시스템의 이론과 데이터 구조

---
## 3.1 CAP 정리

`CAP정리`란 분산 시스템의 기본 원칙으로써 **일관성,가용성,파티션 허용성을 동시에 취할 수 없다**는 것을 의미하며 브루어의 정리라고도 알려져 있다.

하드웨어 고장, 네트워크 장애, 심지어는 정기적인 시스템 점검 같은 여러 가지 불가피
한 이유로 네트워크 파티션이 발생할 수 있다.
이런 상황에서 **일관성과 가용성중 어떤것을 우선시 할지 결정** 해야한다.

1. 가용성과 파티션 허용성(AP)
    - 파티션 발생시 시스템 계속 운영가능 하지만 모든 노드가 동일한 데이터를 유지하기 어려움 

2. 일관성과 파티션 허용성(CP)
    - 모든노드가 항상 같은 데이터를 유지하려고 하므로 파티션 상태에서는 일부 노드가 사용자 요청을 처리하지 못하는 경우 발생

**결론적으로 CAP정리에선 요 3가지간의 균형을 이루는 것이 본질적으로 불가능**

> `일관성` 모든 노드가 동일한 데이터 상태를 유지하는 것
> 
> `가용성` 시스템이 항상 요청을 수용할 수 있는 능력


분산 시스템을 형태가 다양하며, 각 시스템은 목적과 요구사항에 따라 달라질수 있다

예를들어 
- 금융 거래처럼 일관성이 중요할 때는 일관성과 파티션 허용성을 갖춘 시스템이 적합
- 웹 애플리케이션처럼 높은 가용성과 빠른 응답성이 중요할 때는 가용성과 파티션 허용성을
  갖춘 시스템이 적합

CAP 정리를 이해하면 분산 시스템 설계에서 한계점과 해결 방안의 타협점을
찾을 수 있다. 결국 CAP 정리는 특정 선택에 따라 발생하는 기대 비용을 보여 주는 것임

## 3.2 PACELC 정리

`PACELC 정리`는 CAP 정리의 연장선에 있는 개념으로 네트워크 파티션이 발생했을 때 CAP 한계점을
보완하려고 등장

파티션 허용성(**P**artition tolerance):
- 분산 시스템이 네트워크 파티션이나 통신 장애가
  발생했을 때도 정상적으로 기능을 수행하고 서비스를 제공할 수 있는 능력을 의미

가용성(**A**vailability):
- 분산 시스템에 대한 모든 요청이 결국 응답을 받도록 보장하는 것

일관성(**C**onsistency):
- 일관성은 분산 시스템의 모든 노드가 시스템의 현재 상태를 일치시키는 것

그 외(**E**lse):
- 네트워크 파티션이 발생하지 않는 상황을 의미
   이 경우 시스템이 정상적으로 작동하는 상태에서 지연 시간과 일관성 중 하나를 선택해야 한다.
  데이터 요청을 빠르게 처리할지, 아니면 데이터 일관성을 중시하여 최신 정보를 제공할지 결정
  
지연 시간(**L**atency):
- 요청을 시작하고 응답을 받기까지 지연된 시간을 의미

일관성 수준(**C**onsistency level):
- 분산 시스템에서 원하는 일관성의 정도를 나타내며 애플리케이션의 요구 사항이나 시스템 아키텍트가 세운 시스템 설계 방향에 따라 달라질 수 있다

PACELC 정리는 CAP 정리보다 더 넓은 경우의 수를 포함하며 네트워크 파티션이 발생했을 때 가용성
과 일관성 중 하나를 선택해야 한다는 점은 CAP 정리와 동일 

그러나 네트워크 파티션이 없을 때도 지연 시간과 일관성 간에 선택해야 하는 상황이 발생 했을 때 일관성을 선택하면서 지연 시간이 늘어나는 것을 감수할 수도 있고, 반대로 낮은 지연 시간을
선택하면서 일관성을 포기하는 경우가 있을수 있다.

PACELC 정리를 이해하고 있다면 애플리케이션의 구현 요구 사항에 따른 우선순위를
세울 수 있으며 우선순위를 바탕으로 무엇에 더 비중을 두어야 할지 정할수 있다

> 분산 시스템에서는 서버 여러 대가 함께 작업을 수행하는데 이때 **각 서버가 동일한 결정을 내리는 것이 중요**하다 
> 그래서 여러 노드 간에 합의를 이루는 과정이 필수이다
>
>  **합의(consensus)** 란 분산 시스템에서 서로 다른 서버들이 동일한 상태나 결정을 유지하도록 하는 과정
> 
>  각 서버가 독립적으로 작동하다 보면 동일한 데이터에 대해 서로 다른 결정을 내릴 수 있는데 <br>
>  예를 들어  한 서버가 정보를 업데이트하는 동안 다른 서버는 이전 정보를 기준으로 작업을 계속하는 경우 데이터 불일치가 발생하는 경우이다
> 
> 따라서 합의 알고리즘은 **모든 노드가 동일한 결정을 내리도록 동기화하여 데이터 일관성을 유지하고, 
> 시스템의 전반적인 안정성을 높이는 역할**을 한다

### 3.2.1 팩소스 알고리즘 

팩소스 알고리즘은 1990년 레슬리 램포트(Leslie Lamport)가 개발한 합의 알고리즘으로 뛰어난 설계와 성능 덕분에 현대 분산 시스템의 초석이 되었으며, 데이터베이스와 분산 저장 시
스템을 비롯한 다양한 애플리케이션의 기반으로 사용한다

팩소스 알고리즘의 핵심은 **분산 시스템에서 여러 문제가 발생하더라도 하나의 값을 도출할 수 있도록 합의**하는 것이다<br>
합의는 시스템 일관성을 유지하고 신뢰성을 확보하는 데 매우 중요하며 여러 노드가 동일한 데이터를 서로 다른 값으로 업데이트하려는 상황에서 더욱 그렇다.
